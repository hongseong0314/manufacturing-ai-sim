# A공정 문제 정의 (v2 - 최종)

본 문서는 A공정(가공)의 수학적 문제 정의를 최종 확정합니다. 이 정의는 시뮬레이션의 물리 엔진이자, 향후 강화학습 및 최적화 알고리즘 개발의 기준이 됩니다.

## 1. 목표 함수 (Objective Function)

A공정의 운영 목표는 **처리량(Throughput)을 최대화**하고, **재작업(Rework) 및 납기 지연(Tardiness)을 최소화**하는 것입니다. 이는 다음과 같은 다목적 함수 `J`를 최대화하는 문제로 공식화할 수 있습니다.

```
maximize J = w_t * Throughput - w_r * ReworkRate - w_d * Tardiness
```

- `w_t, w_r, w_d`: 각 목표의 중요도를 나타내는 가중치.

## 2. 상태 공간 (State Space, S)

시간 `t`에서의 상태 `S_t`는 A공정 환경의 모든 정보를 포함하며, 스케줄러가 최적의 행동을 결정하기 위한 기반이 됩니다.

- **`M_t` (장비 상태):** 각 장비 `m_i`의 현재 상태 집합.
  - `m_i.status`: 'idle' 또는 'busy'.
  - `m_i.finish_time`: 작업 완료 예상 시각.
  - `m_i.m_age`: 장비의 총 누적 사용량 (영구).
  - `m_i.u`: 현재 부자재의 누적 사용량.
- **`Q_t` (대기열 상태):**
  - `wait_pool`: 신규 Task 대기열.
  - `rework_pool`: 재작업 Task 대기열.
- **`T_t` (시간 정보):**
  - `current_time`: 현재 시뮬레이션 시간.

## 3. 행동 공간 (Action Space, A)

시간 `t`에서의 행동 `A_t`는 유휴 상태('idle')인 각 장비에 대한 작업 할당을 의미합니다.

- `A_t = {machine_id: assignment, ...}`
- `assignment` 딕셔너리는 다음을 포함합니다:
  - `'task_uids'`: 해당 장비에 할당할 Task의 고유 ID 리스트.
  - `'recipe'`: 해당 배치 처리에 사용할 레시피 `[s1, s2, s3]`.

## 4. 전이 함수 (Transition Function) - 물리 모델 (v9)

상태 `S_t`에서 행동 `A_t`를 취했을 때, 다음 상태 `S_{t+1}`로 어떻게 변하는지를 정의합니다. 핵심은 A공정의 품질 `y_a`가 결정되는 방식입니다.

`y_a = g(s, m_age) * E(u) + N(u)`

---

### 4.1. 기본 품질 함수 `g(s, m_age)`

레시피 `s`와 장비 나이 `m_age`로 결정되는 기본 품질. **장비 경시성**과 **비선형 교호작용**을 포함합니다.

`g(s, m_age) = (w1(m_age)*s1 + w2*s2 + w3*s3 + b(m_age)) + w12(m_age)*s1*s2`

- **장비 경시성:**
  - `w1(m_age) = W1_BASE * (1 - DELTA_W1 * m_age)`
  - `w12(m_age) = W12_BASE * (1 - DELTA_W12 * m_age)`
  - `b(m_age) = B_BASE - DELTA_B * m_age`
- **비선형 교호작용:**
  - `w12(m_age)*s1*s2` 항을 통해, `s1`과 `s2`가 서로의 효과에 영향을 미칩니다.

### 4.2. 부자재 효율 함수 `E(u)`

부자재 사용량 `u`에 따라 레시피 효율이 **비선형적으로 감소**합니다 (`tanh` 기반).

`E(u) = 1 - BETA * tanh(BETA_K * u)`

- `u`가 0일 때 효율은 1(100%)이며, `u`가 무한히 커지면 `1 - BETA` 값으로 수렴합니다.

### 4.3. 불확실성(노이즈) 함수 `N(u)`

부자재 사용량 `u`에 따라 공정의 불확실성(분산)이 **비선형적으로 증가**합니다 (`tanh` 기반).

`N(u) ~ Normal(0, σ_u^2)`

- `σ_u = GAMMA * tanh(GAMMA_K * u)`
- 불확실성의 표준편차 `σ_u`는 `u`가 무한히 커져도 최대 `GAMMA` 값으로 수렴합니다.

### 4.4. 파라미터 테이블

| 파라미터 | 현재 값 | 설명 |
|---|---|---|
| `W1_BASE` | 0.5 | s1의 기본 가중치 |
| `W2_BASE` | 0.3 | s2의 기본 가중치 |
| `W3_BASE` | 0.2 | s3의 기본 가중치 |
| `B_BASE` | 45.0 | 기본 품질 상수 (offset) |
| `W12_BASE`| 0.01 | s1, s2 교호작용 기본 가중치 |
| `BETA` | 0.2 | 부자재 효율 감소 최대치 (20%) |
| `BETA_K` | 0.1 | 부자재 효율 감소 곡선의 가파름 |
| `GAMMA` | 1.5 | 품질 노이즈(표준편차)의 최대치 |
| `GAMMA_K` | 0.1 | 품질 노이즈 증가 곡선의 가파름 |
| `DELTA_W1`| 0.001 | w1의 장비 노후화 민감도 |
| `DELTA_W12`| 0.0001| w12의 장비 노후화 민감도 |
| `DELTA_B` | 0.02 | b의 장비 노후화 민감도 |

---

## 5. 제약 조건 및 규칙 (Constraints & Rules)

- **QA (Quality Assurance):** 처리된 Task의 품질 `y_a`는 반드시 `task.spec_a` 범위 내에 있어야 성공.
- **재작업 (Rework):** QA 실패 시, 해당 Task는 `rework_pool`로 이동하여 다음 스케줄링에서 우선적으로 고려됩니다.
- **부자재 교체 (Consumable Replacement):** 현재 `process_a_env.py`에는 `u >= 10`일 경우, 작업 완료 후 자동으로 부자재를 교체하는 규칙이 하드코딩되어 있습니다.
