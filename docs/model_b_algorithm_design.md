# B공정 알고리즘 설계 (v2 - 최종)

본 문서는 B공정(세정)의 Task 할당 및 레시피 선택을 위한 알고리즘들을 정의합니다. 각 알고리즘은 `src/environment/schedulers_b.py`에 구현된 클래스에 해당합니다.

---

## 1. `FIFOBaseline` (Baseline)

가장 간단한 스케줄러로, 다른 모든 지능형 스케줄러의 성능을 비교하기 위한 **성능 최저선(Baseline)** 역할을 합니다.

-   **Task 선택 정책:** `rework_pool`의 Task를 FIFO로 우선 처리하고, 없을 경우 `wait_pool`의 Task를 FIFO로 처리합니다.
-   **Recipe 선택 정책:** **고정 레시피**를 사용합니다. B공정의 상태(`b_age`, `v`)나 Task의 특성을 전혀 고려하지 않고, 항상 사전에 정의된 `default_recipe_B` (예: `[50.0, 50.0, 30.0]`)를 반환합니다.
-   **한계:** 상태 변화에 대응하지 못해, 시간이 지남에 따라 비효율이 발생할 가능성이 높습니다.

---

## 2. `RuleBasedScheduler` (Rule-Based)

`FIFOBaseline`보다 한 단계 발전된 스케줄러로, **장비와 용액의 현재 상태**를 모두 고려하여 레시피를 동적으로 결정합니다.

### 2.1. Task 선택 정책

-   `FIFOBaseline`과 동일 (rework 우선, FIFO).

### 2.2. Recipe 선택 정책

-   **방식:** **2차원 상태 기반 규칙 매트릭스**
-   **로직:**
    1.  **상태 분류:**
        -   **용액 상태 (`v`):** 현재 용액의 사용량 `v`를 기준으로 'fresh', 'medium', 'old' 3단계로 분류합니다.
        -   **장비 상태 (`b_age`):** 장비의 총 사용량 `b_age`를 기준으로 'new', 'medium', 'old' 3단계로 분류합니다.
    2.  **기본 레시피 조회:** (용액 상태, 장비 상태)의 조합, 총 9가지 경우의 수에 대해 미리 정의된 `recipe_library`에서 최적의 기본 레시피를 조회합니다.
        -   예: `('fresh', 'new')` → `[40.0, 45.0, 25.0]` (보수적 레시피)
        -   예: `('old', 'old')` → `[80.0, 70.0, 50.0]` (공격적 레시피)
    3.  **미세 조정:** 조회된 기본 레시피를 Task의 품질 요구사항(`spec_b`)에 맞춰 추가적으로 미세 조정합니다. (예: 요구 스펙이 높으면 레시피를 10% 강화)

### 2.3. 기대 동작

-   장비와 용액의 노후화 상태를 모두 고려하여 레시피를 조정하므로, `FIFOBaseline`에 비해 높은 QA 통과율과 안정성을 보일 것으로 기대됩니다.

## 3. `RLBasedScheduler` (Placeholder)

향후 강화학습(RL) 에이전트가 구현될 것을 대비한 인터페이스입니다. 현재는 `RuleBasedScheduler`의 로직을 그대로 따라가는 **대리(Fallback) 역할**을 수행합니다.

-   **목표:** 장기적인 보상을 극대화하는 최적의 레시피 선택 및 Task 처리 순서 정책 `π(action | state)`을 학습합니다.
-   **상태 공간 (State Space):**
    -   `len(wait_pool)`, `len(rework_pool)`
    -   현재 Task의 `spec_b`, A공정에서 넘어온 품질 `realized_qa_A`
    -   모든 B공정 장비의 `b_age`, `v`, `status`
-   **행동 공간 (Action Space):**
    -   다음에 처리할 Task 선택 (EDD, SPT 등 다양한 규칙 중에서 선택)
    -   레시피 `[r1, r2, r3]` 값 직접 결정 (Continuous) 또는 사전 정의된 레시피 세트 중에서 선택 (Discrete)
-   **보상 함수 (Reward Function):**
    -   `+` B공정 QA 통과
    -   `-` B공정 재세정 발생 (패널티)
    -   `-` C공정 대기 시간 증가에 따른 간접적 패널티
